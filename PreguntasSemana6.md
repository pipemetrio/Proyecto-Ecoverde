# Solución de preguntas sobre el valor de Git y GitHub en DevOps

## 1. ¿Por qué Git ayuda a reducir pérdida de trabajo?

Porque mantiene un historial de cambios que permite recuperar versiones anteriores y revertir errores sin perder el progreso del proyecto.

## 2. ¿Por qué GitHub aporta al trabajo colaborativo?

Porque permite almacenar repositorios en la nube y compartirlos con otros desarrolladores. Además, facilita que varias personas trabajen sobre el mismo proyecto, manteniendo sincronizados los cambios y el historial de trabajo.

## 3. ¿Qué ventaja tiene trabajar con ramas?

Las ramas permiten desarrollar nuevas funcionalidades o realizar pruebas de manera independiente, sin afectar la versión principal del proyecto. Una vez que los cambios están listos, pueden integrarse de forma segura mediante un merge.

## 4. ¿Cómo se relaciona Git con despliegue continuo o integración continua?

Git actúa como fuente de control para los cambios del proyecto. Cada commit o actualización puede activar procesos automáticos de pruebas, integración o despliegue, facilitando la implementación continua de nuevas versiones.

## 5. ¿Qué tipo de deuda técnica puede evitarse usando control de versiones de forma adecuada?

Ayuda a evitar problemas como la falta de trazabilidad, la pérdida de cambios importantes, la duplicación de trabajo y la desorganización entre versiones.

# Solución de preguntas del cierre práctico y consolidación

## 1. ¿Qué comandos Git se usaron con mayor frecuencia?

Los comandos que más se utilizaron fueron:

```bash
git status
git add .
git commit -m
git log --oneline
git switch
git push
```

## 2. ¿Qué diferencia se encontró entre trabajo local y remoto?

El repositorio local es donde se desarrollan y prueban cambios. El remoto sirve para compartir, respaldar y sincronizar esos cambios con otros usuarios.

## 3. ¿Qué parte del flujo con ramas resultó más clara o más difícil?

Lo más facil fue crear las ramas y hacer cambios en ellas, pero lo más dificil fue saber identificar de que forma se logran combinar las ramas entre si.

## 4. ¿Qué valor tienen Git y GitHub para un proyecto DevOps?

Proporcionan trazabilidad, colaboración, automatización y una base para procesos de integración y despliegue continuo.
